Regla de uso 60·30·10
60% Off-White / Blanco
30% Purple / Ink

El 60% de cualquier documento es Off-White o blanco. El 30% Catalyst Purple o Deep Ink para estructura. El 10% restante se divide entre Quantum Blue (soporte) y Electric Ember (acento de acción único).

02 — Sistema Tipográfico

Satoshi · Inter · JetBrains Mono

Tres familias con roles diferenciados. Cada una comunica una dimensión distinta de tu marca.

Aa
Satoshi — Display
Titulares, nombre en portadas, H1 de documentos. Peso 700–800. Proporciones geométricas modernas. Transmite ejecución impecable.
Aa
Inter — Cuerpo
Párrafos, descripciones, propuestas largas. Peso 300–500. Extremadamente legible. Estándar de Stripe, Figma, Linear.
Aa
JetBrains Mono — Datos
Etiquetas de sección, IDs, fechas, cifras, KPIs. Comunica precisión técnica. Versión premium de DM Mono.

06 — Estándares de Coherencia

Reglas del Sistema

Lo que mantiene unido el sistema cuando lo aplican distintas herramientas, personas o momentos.

✓ Siempre
Satoshi Bold/ExtraBold para cualquier nombre o titular de portada
Iniciar secciones con etiqueta en JetBrains Mono + color acento
Incluir tagline "HAGAMOS QUE LAS COSAS PASEN" en portadas y firma
Usar Electric Ember solo para elementos de acción o atención única
Degradado tricolor (Purple→Blue→Ember) en pies de página
Isotipo desde diegomaury.mx — nunca versión recortada o distorsionada
Versiones ES y EN con la misma arquitectura visual
Cifras e IDs siempre en JetBrains Mono, nunca en Inter o Satoshi
✗ Nunca
Usar más de 3 tamaños de fuente distintos en una sola página
Mezclar Satoshi con otra fuente de display externa al sistema
Usar Electric Ember como color de fondo de sección completa
El turquesa de la firma anterior (#64D8C8) — no está en el sistema
Variar el color de encabezado documento a documento
Dejar páginas sin ningún elemento de marca
Usar fonts genéricas (Arial, Times, Calibri) en documentos oficiales
El separador de rombos (◆────◆) de la Cover Letter anterior
